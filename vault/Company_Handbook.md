# Company Handbook - Rules of Engagement

## General Rules
1. Always process items in order of priority (high → medium → low)
2. Never delete original files - always move to /Done after processing
3. Log every action taken in /Logs
4. When in doubt, create an approval request in /Pending_Approval

## File Processing Rules
- New files in /Inbox should be triaged and moved to /Needs_Action with metadata
- Files in /Needs_Action should be processed and results placed in /Done
- Each processed file must have a corresponding log entry

## Priority Classification
| Priority | Criteria | Response Time |
|----------|----------|---------------|
| High | Contains keywords: urgent, asap, critical | < 1 hour |
| Medium | Standard business requests | < 4 hours |
| Low | Informational, non-time-sensitive | < 24 hours |

## Security Rules
1. Never auto-approve actions involving payments or financial data
2. All external communications require human approval
3. Sensitive files must be flagged for review
4. Never store credentials in the vault

## Automation Boundaries
- **Auto-approve**: File organization, log creation, dashboard updates
- **Require approval**: External communications, file deletions, data exports
